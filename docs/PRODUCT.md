# 产品定义

## 产品定位

Task Companion 是 Obsidian 内的本地优先任务辅助插件：以原始 Markdown Tasks 为唯一任务来源，通过明确、可审计的规则选择任务、可靠计时，并在后续阶段逐步形成执行档案和复盘材料。

## 当前可用范围

- Phase 1：插件骨架、设置、测试命令和基础日志。
- Phase 2：25/50 分钟与自定义倒计时、暂停/继续/结束、重载恢复及 `status` 代码块。
- Phase 3：正式任务识别、今日/重点/日常分类、稳定任务 ID、来源跳转和母任务绑定。
- Phase 4：25/50/自由计时和快速推进形成长期执行会话，支持可选进展、当前下一步、失败重试和最小历史查看。

## 设计原则

1. **原始 Tasks 是唯一来源**：不建立第二套封闭待办数据库。
2. **本地优先**：默认不联网，不采集遥测。
3. **显式操作**：任务扫描只在用户打开任务选择器或刷新时发生，不后台轮询。
4. **最小安全写入**：只为正式任务追加稳定块 ID；其他文本保持原样。
5. **规则先行**：每阶段先有验收标准和失败测试，再进入实现。
6. **隔离测试**：自动与人工开发验收仅使用仓库内 `test-vault/`。

## 当前非目标

- 子任务、正式复盘、主页接入和 AI。
- 云同步、外部服务或正式 Vault 安装。

## 核心边界

正式主页保持冻结。开发阶段不访问、挂载、复制或修改正式 Vault；人工验收只打开仓库内测试 Vault。
