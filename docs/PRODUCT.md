# 产品定义

## 产品定位

Task Companion 是 Obsidian 内的本地优先任务辅助插件：以原始 Markdown Tasks 为唯一任务来源，通过明确、可审计的规则选择任务、可靠计时，并在后续阶段逐步形成执行档案和复盘材料。

## 当前可用范围

- Phase 1：插件骨架、设置、测试命令和基础日志。
- Phase 2：25/50 分钟与自定义倒计时、暂停/继续/结束、重载恢复及 `status` 代码块。
- Phase 3：正式任务识别、今日/重点/日常分类、稳定任务 ID、来源跳转和母任务绑定。
- Phase 4：25/50/自由计时和快速推进形成长期执行会话，支持可选进展、当前下一步、失败重试和最小历史查看。
- Phase 5：母任务可自然长出一层子任务，支持状态变化、当前下一步、执行目标绑定和分项投入统计。
- Phase 6：用户明确完成母任务或子任务后进入待复盘队列，生成自动统计并可保存为 Vault 内可读 Markdown。
- Phase 7：Markdown 中可嵌入当前任务、今日、重点、日常和复盘组件；当前任务合并专注状态，提醒卡与当前卡标题共用执行准备。执行准备在同一窗口内完成母/子目标选择、子任务新增和更多操作，不再为拆解重复打开第二个窗口；计时、完成和复盘组成连续且可返回的流程，并使用统一、紧凑、主题自适应的视觉规范。结束计时只保存会话，不能代替任务完成确认。
- Phase 8：已复盘母任务可沉淀为版本化经验模板，新任务只在用户确认后采用模板步骤；插件同时提供版本化本地 API、11 个生命周期事件和默认停用的声明式受控扩展。
- Phase 9：冻结业务功能，制作版本 1.0.0 的桌面端三文件发布包、校验和、安装/升级/回滚说明，并先在仓库测试 Vault 验收。
- Phase 10：插件以只读快照统一测试主页的日常、今日、重点、待推进提醒逻辑；`Home-Test` 保持原视觉与交互，仅替换呈现数据源。

## 设计原则

1. **原始 Tasks 是唯一来源**：不建立第二套封闭待办数据库。
2. **本地优先**：默认不联网，不采集遥测。
3. **显式操作**：任务扫描只由组件加载、用户操作或插件内部状态变化触发，不监听 Vault，不后台轮询。
4. **最小安全写入**：只为正式任务追加稳定块 ID；其他文本保持原样。
5. **规则先行**：每阶段先有验收标准和失败测试，再进入实现。
6. **隔离测试**：自动与人工开发验收仅使用仓库内 `test-vault/`。
7. **状态与删除分离**：取消是可恢复的停用，永久删除是用户明确触发的目标级数据清理，不能用取消暗中代替删除。
8. **建议不等于修改**：经验模板只能提出建议，未经用户明确采用不得新增或改写任务。
9. **扩展能力可证明地受限**：用户目录不执行任意代码，只解释已校验的声明式动作；危险权限不存在，而不是仅靠提示禁止。

## 当前非目标

- 多层子任务、依赖图、复盘表单模板、未经确认的正式主页安装和 AI 自动修改。
- 任意 JavaScript、脚本市场、网络、Shell 或任意 Vault 文件操作。
- 云同步、外部服务或正式 Vault 安装。

## 核心边界

原桌面主页、移动主页和原 CSS 保持冻结。用户已单独授权 `Home/Home-Test.md` 接入插件只读快照；该授权不扩展到其他主页或样式文件。
