# 产品定义（第一版草案）

## 产品定位

Task Companion 计划作为 Obsidian 内的本地优先任务辅助插件，帮助用户在笔记环境中以明确、可审计的规则管理任务。当前仅确定开发边界，具体交互和任务格式尚未确认。

## 本轮目标

- 建立隔离、可重复的插件开发环境。
- 建立产品规则、数据模型、路线图、状态与验收文档。
- 建立可构建、可类型检查、可 lint、可测试的最小骨架。

## 非目标

- 不实现任务采集、展示、编辑、同步或提醒。
- 不实现计时器、番茄钟、倒计时、后台轮询或周期调度。
- 不接入任何真实任务或正式 Vault。
- 不进行安装、发布、遥测、联网或外部服务集成。

## 设计原则

1. **本地优先**：默认不联网，数据留在用户明确授权的 Vault 范围内。
2. **显式授权**：扫描、解析和写入的范围必须由用户确认。
3. **可逆与可审计**：后续写操作应可追踪、可测试，并尽量可撤销。
4. **规则先行**：任务语义和冲突处理先形成文档与验收标准，再开发功能。
5. **隔离测试**：开发与测试只使用 `test-vault/` 中的人造数据。

## 初步用户价值假设

用户希望在不离开 Obsidian 的情况下理解和整理笔记中的任务，同时保留对文件、格式和变更的控制权。该假设需要用户确认与后续需求访谈验证。
