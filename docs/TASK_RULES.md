# 任务规则

## 1. 原始任务与正式任务

原始任务继续使用 Markdown Tasks。只有未完成、未取消且包含 `⏫`、`🔼`、`🔽`、`⏬` 或 `🔁` 的任务才进入 Task Companion。

以下内容必须排除：

- `[x]`、`[X]` 已完成任务；
- `[-]` 已取消任务；
- 文本中明确包含 `cancelled`、`canceled` 或“取消”的任务；
- 没有优先级或循环标记的普通复选框。

## 2. 今日待办

任务未完成、未取消且属于正式任务，并符合以下任一条件时进入今日待办：

1. 同时有 Start（🛫）和 Due（📅），且 `Start ≤ 今天 ≤ Due`；
2. Scheduled（⏳）是今天；
3. Due 是今天；
4. Due 早于今天且仍未完成，标记为“逾期”；
5. 只有 Start、没有 Due，且 `Start ≤ 今天`；
6. 只有未来 Due、没有 Start，在截止前不进入；
7. 没有任何日期，不进入今日待办。

Scheduled 等于今天和已逾期 Due 优先成立，即使其他日期组合不一致，也不能隐藏需要处理的任务。

## 3. 重点与日常任务

- 所有活动的 `⏫` 任务都是重点任务，不受日期限制。
- 同时属于今日与重点的任务只显示一次，分类为“今日待办＋重点任务”。
- 活动的 `🔁` 任务属于日常任务；是否进入今日仍由日期规则决定。

## 4. 稳定任务 ID

首次安全扫描正式任务时，在任务行末尾追加六位小写十六进制块 ID：

```markdown
^tc-a7f3d2
```

规则：

- 已有有效 `^tc-xxxxxx` 保持不变；
- 已有其他块 ID 时保持原文并报告冲突；
- 同一扫描中 ID 必须唯一；重复 ID 的后续任务不进入选择结果；
- 写入前比较完整文件和目标原行；任何冲突或写入失败都保持原文；
- 不写入番茄图标、番茄次数或其他显示字段。

## 5. 扫描和显示

- 只扫描当前用户明确打开任务选择器的 Vault 中的 Markdown 文件。
- 开发与验收只允许仓库内 `test-vault/`。
- 扫描失败显示数量汇总，不在日志或通知中暴露任务正文。
- 单个文件失败不阻断其他文件。
- 选择器显示任务正文、分类、日期和 Vault 相对来源路径，并支持显式刷新。
