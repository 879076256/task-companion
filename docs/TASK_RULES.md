# 任务规则（第一版草案）

> 本文只定义候选规则和决策点，不代表已经实现。

## 已确定边界

- 当前仓库不保存真实任务。
- 当前插件不扫描、不解析、不创建、不修改任何任务。
- 后续测试任务必须是 `test-vault/` 内的人造内容。
- 计时器及任何周期性执行机制不在当前范围内。

## 候选任务语义

后续若采用 Markdown task，最小候选形式为 `- [ ] 文本`，完成态候选形式为 `- [x] 文本`。以下内容均未确认：

- 是否支持大写 `X`、自定义状态字符或非列表任务。
- 截止日期、计划日期、优先级、标签、项目和上下文的语法。
- 重复任务、子任务、依赖关系和归档规则。
- 完成任务时是否写入完成时间，以及写入格式。

## 候选处理原则

- 只处理用户明确授权范围内的 Markdown 文件。
- 无法无歧义解析的内容保持原样，并给出可理解的诊断。
- 默认不重排用户文本，不覆盖未知元数据。
- 写入前后应能验证目标行和预期变更，冲突时停止写入。
- 同一任务的稳定标识、去重规则与跨文件移动语义尚待确认。
