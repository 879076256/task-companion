# 决策记录

## 2026-07-16：Phase 2 可靠计时器

- 用户明确批准 Phase 2 仅实现可靠计时器与 `status` 嵌入组件。
- 自由计时定义为用户输入分钟数的自定义倒计时，以保持“剩余时间”语义一致。
- 核心状态机使用绝对毫秒时间戳，不依赖 tick 次数累计时间。
- 暂停态显式持久化剩余秒数；重载不消耗暂停时长。
- 同时最多一个运行或暂停会话；重复启动必须被拒绝。
- 计时短期状态仅保存到插件 `data.json`，不写入笔记。
- 系统通知和声音是可选外围能力，失败不得改变核心计时状态。
- 本阶段不实现真实任务选择、Tasks 解析、任务层级、执行记录、复盘和主页集成。

## 2026-07-16：Phase 4 执行会话重做

- 每次计时沿用启动时 UUID，快速推进单独生成 UUID；都关联稳定 taskId。
- 正常倒计时活动时长等于计划时长；提前结束按墙钟时间扣除累计暂停时间。
- 会话结束先进入 `data.json` 待写队列，表单保存、跳过或关闭后再追加 Vault 日志；意外重载可恢复基础会话。
- 长期格式为 `TaskCompanion/Sessions/YYYY-MM.jsonl`，每行一条完整 schemaVersion 1 会话；读取器支持 v0 迁移。
- 追加前检查 sessionId，避免“已写入但队列清理失败”造成重复记录。
- 当前下一步由该 taskId 最新非空 `nextAction` 派生，不建立第二套任务数据库。
- 旧错误 Phase 3/4 分支和标签已按用户要求直接删除；只保留正确 `v0.3.1-task-selection` 基线。

## 2026-07-16：Phase 5 一层任务拆解

- 子任务不是 Markdown Tasks 的第二来源；它是稳定 taskId 下的执行拆解档案，严格只有一层。

- 当前下一步优先显示活动子任务标题；没有子任务下一步时回退 Phase 4 的文字 nextAction。

## 2026-07-16：Phase 7.4 完成确认与复盘资格

- 计时会话结束与任务完成是两件独立的事：自然完成或提前结束计时只保存 ExecutionSession，不改变母任务/子任务状态，也不打开完成收尾表单。
- 只有用户明确勾选或点击当前任务卡完成按钮，目标才进入待复盘。母任务即使没有执行档案也生成零统计的 pending ReviewEvent，避免“已完成却找不到复盘”的不一致。
- 当前任务卡的完成目标由当前执行绑定决定：活动 subtaskId 优先，否则完成母任务；成功后用一条零时长 quick ExecutionSession 保存可选的完成、下一步和注意事项文字。
- 今日/重点/日常、任务选择器和 pending 复盘统一使用标题前复选框；当前任务卡是唯一保留独立完成图标的主页卡片。
- pending 反勾选不删除追加式 ReviewEvent：母任务按稳定块 ID 恢复 `[ ]`，子任务追加 `reopened`，资格检查将原 pending 隐藏；重新完成复用原 pending reviewId。completed 复盘不因返工删除，明确永久删除子任务时例外。
- `blockerReason` 字段名保持不变以兼容现有 schemaVersion 2 日志，界面统一改称“注意事项”。

## 2026-07-17：取消与永久删除子任务分离

- 用户确认“取消”表示停止执行但保留档案，“删除”表示彻底去掉该子任务的直接记录；本决策取代 2026-07-16 Phase 7.1 中“仅取消后软删除并保留审计事件”的限制。
- 更多菜单对 active、cancelled、completed 子任务统一显示“删除子任务”，无需先取消；取消仍保留恢复入口。
- 永久删除按 `taskId + subtaskId` 物理清理子任务事件引用、长期与待写 ExecutionSession、子任务复盘索引/队列和可读 Markdown。父任务级复盘是独立归档，兄弟子任务和损坏 JSONL 行不得被连带删除。
- 删除正在计时的目标先直接重置，不结束会话、不弹收尾表单；目标的累计投入随历史会话删除归零。
- 会话与复盘先清理，子任务档案最后清理；这样发生写入失败时仍保留可见子任务供重试。删除成功后才提示完成。
- 当前任务卡紫色目标框中的母任务标题复用提醒卡标题的执行准备流程；重新选择子任务后由既有 TimerService 通知刷新为母任务、子任务两行，不新增第二套选择或显示状态。
- 执行准备移除“编辑任务拆解”：同一窗口列出全部子任务，活动项可选择，已完成/已取消项不可选择但保留恢复或删除菜单；新增框固定在列表下方。这个单窗口流程复用 SubtaskService 和原有删除清理回调，不建立第二份拆解状态。
- 当前任务卡在子任务行增加与母任务完成键对齐的完成入口。子任务按钮调用 `completeCurrentTarget` 完成当前活动子任务；母任务按钮调用独立的父级入口完成母任务并执行未完成子任务分流。两者都复用既有完成服务和收尾表单，不在组件内复制写入逻辑。
- 当前任务卡的计时、会话、执行目标和子任务已有内部通知，因此移除重复的手动刷新按钮；提醒与复盘组件仍保留刷新以显式重扫 Markdown，避免为此引入 Vault 后台监听。
