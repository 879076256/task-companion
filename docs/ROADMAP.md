# 路线图

## Phase 0：安全环境、架构与文档（已完成）

独立仓库、测试 Vault、长期文档和安全边界已建立。

## Phase 1：插件骨架与工程基线（已完成）

插件生命周期、设置、测试命令、Modal、日志和质量命令已建立。

## Phase 2：可靠计时器与状态组件（已完成）

用户已确认 Phase 2 完成；计时状态机、重载恢复、控制 Modal 和 `status` 代码块保留。

## Phase 3：正式任务识别与任务选择（已完成）

- 正式任务过滤与七条今日规则。
- 重点、今日＋重点和日常任务分类去重。
- 稳定块 ID、安全冲突检测与失败隔离。
- 支持搜索、刷新、来源跳转和母任务绑定的选择 Modal。
- 当前 taskId 与短期计时状态持久化。
- 20 个单元/结构测试及测试 Vault 构建校验通过。
- 测试 Vault 人工验收通过：来源跳转、任务绑定、计时控制重开与切换保护正常。

## Phase 4：执行会话、进展记录与当前下一步（已完成并通过人工验收）

- 25/50/自由计时和快速推进生成唯一 ExecutionSession。
- 记录活动/暂停时间、正常/提前结束和可选进展字段。
- 按月 JSONL 追加日志、旧格式读取迁移、待写队列与幂等重试。
- 最新 nextAction 回显和当前任务最小会话历史。
- 自动检查及仓库测试 Vault 人工验收均已通过。

## Phase 5：一层任务拆解与进度（已完成并通过人工验收）

- 支持一层子任务添加、改名、排序、完成、取消、返工及当前下一步。
- 变化按母任务写入可查看、可备份的追加式 JSONL 档案。
- 计时和快速推进可绑定母任务或活动子任务。
- 统计子任务完成数、分项投入、母任务直接投入、总次数和总时长。
- 不显示虚假百分比，不包含多层树、依赖、甘特图、AI 或模板界面。
