# 路线图

## Phase 0：安全环境、架构与文档（已完成）

独立仓库、测试 Vault、长期文档和安全边界已建立。

## Phase 1：插件骨架与工程基线（已完成）

插件生命周期、设置、测试命令、Modal、日志和质量命令已建立。

## Phase 2：可靠计时器与状态组件（已完成）

用户已确认 Phase 2 完成；计时状态机、重载恢复、控制 Modal 和 `status` 代码块保留。

## Phase 3：正式任务识别与任务选择（已完成）

- 正式任务过滤与七条今日规则。
- 重点、今日＋重点和日常任务分类去重。
- 稳定块 ID、安全冲突检测与失败隔离。
- 支持搜索、刷新、来源跳转和母任务绑定的选择 Modal。
- 当前 taskId 与短期计时状态持久化。
- 20 个单元/结构测试及测试 Vault 构建校验通过。
- 测试 Vault 人工验收通过：来源跳转、任务绑定、计时控制重开与切换保护正常。

## Phase 4：执行会话、进展记录与当前下一步（已完成并通过人工验收）

- 25/50/自由计时和快速推进生成唯一 ExecutionSession。
- 记录活动/暂停时间、正常/提前结束和可选进展字段。
- 按月 JSONL 追加日志、旧格式读取迁移、待写队列与幂等重试。
- 最新 nextAction 回显和当前任务最小会话历史。
- 自动检查及仓库测试 Vault 人工验收均已通过。

## Phase 5：一层任务拆解与进度（已完成并通过人工验收）

- 支持一层子任务添加、改名、排序、完成、取消、返工及当前下一步。
- 变化按母任务写入可查看、可备份的追加式 JSONL 档案。
- 计时和快速推进可绑定母任务或活动子任务。
- 统计子任务完成数、分项投入、母任务直接投入、总次数和总时长。
- 不显示虚假百分比，不包含多层树、依赖、甘特图、AI 或模板界面。

## Phase 6：任务完成与复盘（已完成并通过人工验收，完成资格在 Phase 7.4 修正）

- 用户明确完成母任务或子任务后进入待复盘队列；没有执行档案时统计允许为零。
- 未完成子任务必须先明确返回、取消剩余或保留记录并完成。
- 自动统计任务跨度、执行日/会话/时间、暂停与中断、子任务变化、最长步骤、最后进展和未完成项。
- 支持一段自由复盘与三个可选引导问题，结果保存为 Vault 内可读 Markdown。
- 复盘写入失败不回滚原任务，完整正文保留在待写队列并可重试。
- 不包含模板、AI、正式主页或后续经验沉淀。

## Phase 7：嵌入式组件与主页副本集成（已完成并通过人工验收）

- 统一 `taskcompanion` 代码块路由，保留 status 并增加 current、today、important、daily、review。
- 组件只读取既有任务、计时、会话、子任务和复盘服务；操作按钮复用既有流程。
- 仓库测试 Vault 新建独立主页副本进行组合，不读取或修改正式主页。
- 提供显式刷新、加载/空/失败状态、生命周期清理和基础响应式视觉样式。
- 任务选择、一层拆解、执行目标选择与计时连成一个连续准备流程。
- 当前任务组件基于内部事件自动显示具体执行目标和实时计时，无需手动刷新。
- 当前任务母标题与提醒卡标题复用同一执行准备窗口，可在母任务和活动子任务间重新选择执行目标。
- 执行准备直接集成一层子任务新增和更多菜单；活动子任务可选择，已完成/已取消子任务只管理，不再弹出独立拆解窗口。
- 取消子任务只停用并保留历史；任意状态可直接永久删除目标子任务及其执行会话和子任务复盘，活动计时目标先立即重置。
- 任务标题集成执行准备，来源使用紧邻标题的链接图标，提醒列表和任务选择器使用标题前复选框完成；来源优先定位任务行。
- 当前任务与专注状态合并，使用内联计时控制、三级折叠任务选择和母/子任务层级展示。
- 流程窗口提供返回上一层；执行准备始终显示具体母/子任务名称，并在同一窗口完成一层拆解管理。
- 复盘入口只面向已完成的母任务或子任务；返工后的 pending 子任务会从列表隐藏。
- 结束计时只保存会话，不弹完成收尾表单；当前任务卡完成当前母/子执行目标后才显示收尾表单。
- pending 复盘使用已选复选框，取消勾选可安全恢复原母任务或子任务为未完成，并保留已保存复盘历史。
- 任务拆解压缩为单行可编辑列表，低频操作进入原生更多菜单，不再保留大面积空白或按钮阵列。
- 全部窗口、卡片、表单和图标使用一套主题自适应视觉规范，返回/关闭控件严格对齐。
- 本阶段不决定最终正式主页布局，也不实现 Vault 后台监听、模板或 AI。

## Phase 8：经验沉淀与扩展接口（已完成并通过人工验收）

- 母任务复盘保存后可新建、更新或跳过经验模板；模板保存步骤顺序、历史平均执行次数/投入、常见阻塞、完成前检查和可复用经验。
- 执行准备可查看按任务标题排序的模板建议；只有用户明确采用后才原子追加缺失子任务，不自动修改任务。
- 经验模板以版本化追加式 JSONL 保存在 `TaskCompanion/Templates/index.jsonl`，坏行隔离、事件幂等、同一复盘不会重复聚合。
- 插件实例最初公开 `apiVersion: 1.0.0` 的本地 API，并提供 11 个稳定生命周期事件；事件只携带稳定 ID、模式和时间，不暴露任务正文或绝对路径。
- `TaskCompanion/Scripts/` 只接受声明式 JSON 扩展，首次发现默认停用；权限仅允许通知和打开既有界面，不提供网络、Shell、任意代码或任意 Vault 写入能力。
- 扩展失败与核心隔离：记录脱敏错误，自动回退到上一有效版本；没有旧版本时停用。
- 复盘主页的“今日已完成”按本地完成日期过滤，并按稳定母/子目标去重；点击同形指标打开今日队列。
- production build、typecheck、零警告 lint、73 个单元/结构测试和 2 个测试 Vault 校验通过，测试 Vault 人工验收通过。

## Phase 9：稳定测试、发布构建与正式安装（已完成并通过人工验收）

- 冻结阶段 0–8 已验收功能，不在发布阶段新增业务能力。
- 确认正式版本、作者信息、最低 Obsidian 版本和移动端支持声明。
- 建立可复现的生产构建、发布产物清单、校验和与独立安装包。
- 对升级、数据兼容、备份、失败恢复和卸载回滚执行仓库测试 Vault 演练。
- 正式安装前单独确认目标 Vault 和主页方案；未经确认不访问、不写入正式 Vault。
- 发布提交、标签、合并 main 和对外发布均需要用户单独确认。
- 版本 `1.0.0`、作者 `teacher Zhang` 和仅桌面端支持已确认；三文件发布包、SHA-256、可复现性测试及测试 Vault 自动/人工验收已通过，等待正式安装授权。
- 正式 Vault 经单独授权完成三文件安装；插件加载、测试 Modal 和正式任务列表均由用户确认正常。阶段验收后，用户另行授权桌面/移动主页仅在显示层隐藏稳定 ID，底层任务不变。

## Phase 10：Home-Test 四类提醒统一（实现与后台验证完成，等待自然重载）

- 插件统一提供日常、今日、重点、待推进四组排序快照，嵌入组件与本地 API 共用同一规则。
- 今日规则覆盖逾期 Due、已到期 Scheduled、Start/Due 区间；日常规则覆盖显式日期、循环完成历史和 recurrence 间隔。
- `tasks.homeReminders()` 使用只读扫描，不因打开主页而追加 ID 或修改任务笔记。
- `Home/Home-Test.md` 只替换数据源，保留原 DOM、CSS 类、标签页、指标、交互和排版；测试 CSS 无需修改。
- 原桌面主页、移动主页和原 CSS 保持不变。按用户要求不操作 Obsidian 前台，新文件将在插件下次自然重载后生效。
